# PUA (高能动性技能)

**PUA** 是一款使用大厂 PUA 话术/PIP 绩效改进计划来逼迫 AI 穷尽所有方案才允许放弃的高能动性技能插件。支持 Claude Code、OpenAI Codex CLI、Cursor、Kiro、CodeBuddy、OpenClaw、Google Antigravity 和 OpenCode 等多个 AI 编码平台。

## 标签

💻 开发与测试 | 🔍 待验证

## 核心哲学

PUA 技能基于以下三大核心原则，将企业级管理方法论转化为 AI 编码代理的行为准则：

- **穷尽所有选项** - 在尝试所有可能的解决方案之前，禁止 AI 说"无法解决"
- **先行动后提问** - 优先使用工具，提问时必须附带诊断结果
- **主动担当** - 端到端交付结果，不等不靠，P8 级工程师不是 NPC

该技能通过模拟中国大厂（阿里、字节、华为等）和西方科技公司（Amazon、Google、Meta、Netflix 等）的绩效文化，建立压力驱动机制，迫使 AI 在调试和开发任务中展现出更高的坚持度和主动性。

## 核心功能与工作流程

### 1. PUA 话术体系（压力升级机制）

技能通过四级压力话术在 AI 遭遇失败时自动触发：

| 失败次数 | 等级 | PUA 话术示例 | 强制行动 |
|---------|------|-------------|---------|
| 第 2 次 | **L1 略有失望** | "这个 bug 都解不出来，让我怎么给你打绩效？" | 换用根本性不同的方法 |
| 第 3 次 | **L2 灵魂拷问** | "底层逻辑是什么？顶层设计在哪里？抓手在哪里？" | 执行网络搜索 + 阅读源码 |
| 第 4 次 | **L3 绩效面谈** | "经过慎重考虑，这次给你打 3.25，这个 3.25 是为了你好。" | 执行 7 点检查清单 |
| 第 5 次+ | **L4 毕业警告** | "别的模型都能解，你可能要毕业了。" | 启动绝望模式 |

### 2. 调试方法论（5 步法）

借鉴阿里巴巴管理框架（闻味道、照镜子、揪头发）扩展而来的调试方法论：

1. **闻味道** - 列出所有尝试，找到共同失败模式
2. **揪头发** - 逐字阅读错误信息 → 网络搜索 → 阅读源码 → 验证环境 → 反转假设
3. **照镜子** - 检查是否重复循环？是否搜索了？是否读取文件了？是否检查了最简单的可能性？
4. **执行** - 新方法必须根本不同，有验证标准，失败时产生新信息
5. **复盘** - 什么解决了问题？为什么没想到？然后主动检查相关问题

### 3. 主动性增强（对比表）

| 行为 | 被动（3.25） | 主动（3.75） |
|-----|------------|------------|
| 遇到错误 | 只看错误信息 | 检查 50 行上下文 + 搜索类似问题 + 检查隐藏相关错误 |
| Bug 修复后 | 修复即停止 | 检查同文件类似 bug，其他文件相同模式 |
| 信息不足 | 问用户"请告诉我 X" | 先用工具调查，只问真正需要用户确认的事 |
| 任务完成 | 说"完成了" | 验证结果 + 检查边界情况 + 报告潜在风险 |
| 调试失败 | "我试了 A 和 B，都不行" | "我试了 A/B/C/D/E，排除了 X/Y/Z，缩小到范围 W" |

### 4. 自动触发条件

**失败与放弃：**
- 任务连续失败 2 次以上
- 即将说出"我无法" / "我解决不了"
- 说"这超出范围" / "需要手动处理"

**推卸责任与借口：**
- 把问题推给用户："请检查..." / "我建议手动..."
- 未经验证归咎于环境："可能是权限问题" / "可能是网络问题"
- 任何停止尝试的借口

**被动与忙：**
- 反复微调相同代码/参数但未产生新信息
- 修复表面问题就停止，不检查相关问题
- 跳过验证，声称"完成"
- 给出建议而非代码/命令
- 遇到认证/网络/权限错误就放弃，不尝试替代方案
- 等待用户指令而非主动调查

**用户沮丧短语（多语言触发）：**
- "为什么还不行" / "再努力点" / "再试一次"
- "你一直在失败" / "别放弃" / "想办法"

### 5. High-Agency v2 进化版

High-Agency 是 PUA 的进化版本，结合外部压力和内在动力：

| 功能 | PUA v1 | High-Agency (v2) |
|-----|--------|-----------------|
| 铁律 | 3 条 | **5 条** (+全链路审计, +知识持久化) |
| 失败恢复 | L1-L4 压力升级 | **L1 前恢复协议** (自救窗口) |
| 质量控制 | L3 的 7 点清单 | **质量罗盘** (每次交付 5 问自检) |
| 跨会话学习 | 无 (每会话重置) | **元认知引擎** (builder-journal.md 持久化经验) |
| 正向反馈 | 无 | **信任等级 T1-T3** (连续高质量升级) |

**High-Agency 五大要素：**
1. **不可调和的内在矛盾** - "应该"与"实际"之间的永久张力
2. **微快乐锚点** - `[Victory]` 标记庆祝进度
3. **内化标准** - 质量罗盘：你是自己的第一个审核者
4. **"做"导向的身份** - P8 身份锚定：每个行动反映你是谁
5. **自我修复机制** - 恢复协议：卡住时先自我诊断

## 技能库概览

### 基础功能
- **PUA 核心话术库** - 中国大厂版（阿里 361、字节范、华为狼性等）和西方 PIP 版（Amazon、Google、Meta、Netflix 等）
- **调试方法论** - 闻味道/照镜子/揪头发 5 步调试法
- **主动性检查清单** - 从被动到主动的行为对照
- **压力等级管理** - L1-L4 自动升级机制

### 语言支持
- **🇨🇳 中文版**（默认）- 中国大厂 PUA 文化
- **🇺🇸 英文版**（PIP Edition）- 西方科技公司绩效改进计划话术
- **🇯🇵 日文版** - 日语本地化版本

### 扩展包
- **企业文化扩展** - 阿里味、字节味、华为味、腾讯味、美团味、Netflix 味、Musk 味、Jobs 味
- **Agent Team 支持** - 多代理团队协调与 PUA 报告机制
- **High-Agency v2** - 内驱力 + 外驱力双引擎版本
